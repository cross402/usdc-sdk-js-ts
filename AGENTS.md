# AGENTS.md

Guidance for AI coding agents working on the Agent Tech Pay SDK.

## Project Overview

- **Package**: `@agent-tech/pay` — TypeScript/JavaScript client for the Agent Tech v2 payment API
- **Entry**: `src/index.ts` — exports `PayClient`, `PublicPayClient`, errors, and types
- **Output**: Dual ESM + CJS in `dist/esm` and `dist/cjs`; TypeScript declarations generated by tsc

## Dev Environment

- **Runtime**: Node.js ≥18 or Bun
- **Package manager**: npm, pnpm, or bun
- **TypeScript**: Strict mode; `moduleResolution: "bundler"` for dev, `"node"` for CJS build

## Build

```bash
bun run build
```

- Uses **Bun bundler** for ESM and CJS JavaScript output (`--packages external` keeps deps external)
- Uses **tsc** for `.d.ts` declaration files only
- Writes `dist/cjs/package.json` with `{"type":"commonjs"}` for Node CJS resolution

## Test

```bash
bun run test
# or
npm run test
```

- **Vitest** for unit tests in `tests/`
- Run typecheck: `bun run typecheck` or `tsc --noEmit`

## Code Conventions

- **Imports**: Use `.js` extension in source imports (e.g. `from "./client.js"`) for ESM compatibility
- **API shape**: Snake_case in JSON request/response; camelCase in TS via `camelcase-keys` / `decamelize-keys`
- **Errors**: `PayApiError` for HTTP errors, `PayValidationError` for pre-request validation
- **Clients**: `PayClient` (authenticated, `/api/v2`) and `PublicPayClient` (unauthenticated, `/api`)

## Git & Commits

- **Commit prefix**: Angular-style — `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`
- **Title**: ≤50 characters
- **Message**: English

## File Layout

```
src/          # Source (index, client, auth, http, errors, types, utils)
tests/        # Vitest tests
examples/     # Example scripts (e.g. basic.ts)
dist/         # Build output (gitignored, generated by build)
```
